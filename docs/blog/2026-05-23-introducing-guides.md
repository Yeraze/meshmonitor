MeshMonitor's blog has a new category: **Guides**. These are longer-form, community-flavored write-ups — how-tos, field notes, lessons learned from real deployments. Not release notes, not changelogs. Just useful operator knowledge that doesn't fit anywhere else.

Two are already up:

- [MeshMonitor in a Hurricane](/blog/2026-05-23-hurricane-preparedness) — operating a mesh through a major storm, from pre-season prep to post-storm recovery.
- [Build an Offline Emergency Kit](/blog/2026-05-23-offline-emergency-kit) — the hardware/compose/tile-server companion: how to put together a self-contained MeshMonitor box that boots cold with its own map.

If you've ever wondered how your nodes will behave when the power's out for four days — or how to build the box you'd want next to you when it happens — start there.

Have a guide of your own? Open a PR against `docs/blog/` with a new dated file and `category: guide` in the frontmatter — solar deployments, antenna tuning, off-grid power, MQTT bridging adventures, anything that helped you and might help someone else. See [CONTRIBUTING.md](https://github.com/Yeraze/meshmonitor/blob/main/CONTRIBUTING.md) for the basics. The mesh is bigger than any one operator; the docs should be too.
